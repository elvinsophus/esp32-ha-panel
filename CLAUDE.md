This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

ESP32-based Home Assistant control panel targeting the **Waveshare ESP32-S3-Touch-LCD-4B** board, built with ESP-IDF and LVGL v9.2.

## Toolchain

- **Framework**: ESP-IDF v6, installed at `C:\esp\v6.0.1\esp-idf`
- **IDE**: VS Code with the ESP-IDF extension

### Common commands

```bash
# Source IDF environment (once per shell session)
. C:/esp/v6.0.1/esp-idf/export.sh

# First-time: install managed components
idf.py update-dependencies

# Build
idf.py build

# Flash and monitor
idf.py -p PORT flash monitor

# Save current sdkconfig as new defaults
idf.py save-defconfig
```

## Architecture

### Component dependencies (`main/idf_component.yml`)

| Component | Role |
|-----------|------|
| `waveshare/esp32_s3_touch_lcd_4b` | BSP — initialises display, touch, and peripherals |
| `lvgl/lvgl` `9.2.0` | UI framework |

The BSP exposes `bsp/esp-bsp.h` and `bsp/display.h`. Call `bsp_display_start()` before any LVGL calls, and wrap all LVGL API calls between `bsp_display_lock(0)` / `bsp_display_unlock()`.

### HAPanel boot UI

The firmware no longer compiles LVGL demo sources. `app_main` starts the
Waveshare BSP display stack and then calls `hapanel_boot_ui_show()` from
`main/ui/` to render the foundation status screen. Keep LVGL calls wrapped by
`bsp_display_lock(0)` / `bsp_display_unlock()`.

### Key hardware configuration (`sdkconfig.defaults`)

- Flash: 16 MB QIO
- PSRAM: Octal 80 MHz, with instructions and rodata fetched from PSRAM
- CPU: 240 MHz
- FreeRTOS tick: 1000 Hz
- LVGL: FreeRTOS OS backend, 2 SW draw units, fast-mem in IRAM
- LVGL demos: disabled; HAPanel owns the first screen

### Clangd / IntelliSense

`.clangd` strips `-f*` and `-m*` compiler flags so clangd does not choke on GCC-specific options. The `compile_commands.json` generated by `idf.py build` is the compilation database.
